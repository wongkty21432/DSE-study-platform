資訊處理 (一) — 資訊處理簡介與數據組織

資訊系統的組成、資訊處理過程、數據與資訊的分別、數據組織與數據控制

📋

資訊系統與資訊處理

▼

資訊系統的組成

資訊系統由以下部分組成：

  • 硬件 (Hardware)：電腦、伺服器、網絡設備等實體設備
  • 軟件 (Software)：操作系統、應用程式、數據庫管理系統
  • 數據 (Data)：原始事實和數字，未經處理
  • 人員 (People)：使用者、管理員、開發人員
  • 程序 (Procedures)：操作流程、政策、規則

資訊處理過程 (IPO模型)

輸入 (Input) → 處理 (Process) → 輸出 (Output) → 儲存 (Storage) → 反饋 (Feedback)

💡 考試提示：IPO模型是資訊處理的核心概念。注意辨別每個階段在具體情境中的對應。
📊

數據與資訊的分別

▼
特性數據 (Data)資訊 (Information)
定義原始事實和數字經處理和組織後有意義的數據
形式無上下文、未經處理有上下文、有結構
用處作為處理的原材料用於決策和分析
例子每個學生的分數：85, 92, 78全班平均分：85，排名：第2名

💡 記憶技巧：數據是「原材料」，資訊是「製成品」。就像麵粉（數據）經烘焙後變成麵包（資訊）。

🗂️

數據層次 (Data Hierarchy)

▼

從最小到最大的數據組織層次：

  1. 字段 (Field)：數據的最小單位，代表一個屬性（如：姓名、年齡）
  2. 記錄 (Record)：一組相關字段的集合（如：一個學生的所有資料）
  3. 檔案 (File)：一組相關記錄的集合（如：全班學生的資料）
  4. 數據庫 (Database)：一組相關檔案的集合（如：全校學生的資料）

字段 → 記錄 → 檔案 → 數據庫

✅

數據驗證 (Data Validation)

▼

確保輸入數據的準確性和有效性：

  • 範圍檢查 (Range Check)：數據是否在允許範圍內（如：年齡0-150）
  • 類型檢查 (Type Check)：數據類型是否正確（如：數字/文字）
  • 長度檢查 (Length Check)：數據長度是否符合要求
  • 格式檢查 (Format Check)：數據格式是否正確（如：日期格式）
  • 存在檢查 (Presence Check)：必填字段是否已填寫
  • 奇偶檢查 (Parity Check)：檢測數據傳輸中的錯誤

數制轉換 (Number Systems)

▼

常用數制

數制基數使用的符號用途
二進制 (Binary)20, 1電腦內部運算
十進制 (Decimal)100-9人類日常使用
十六進制 (Hexadecimal)160-9, A-F簡潔表示二進制、顏色代碼、記憶體地址

轉換方法

十進制 → 二進制：重複除法（除以2，收集餘數，從下往上讀取）

二進制 → 十進制：每位乘以2的冪次後求和

十六進制 → 二進制：每個十六進制位轉為4位二進制

例：2F₁₆ = 0010 1111₂ → 47₁₀

字符編碼 (Character Encoding)

▼
編碼標準位元數字符數支援語言
ASCII7-bit (擴展8-bit)128 (256)基本英文
Big-516-bit~13,000繁體中文
GB16-bit~7,000簡體中文
Unicode (UTF-8)8-32 bit>143,000幾乎所有語言
💡 考試重點：UTF-8與ASCII相容（頭128個字符相同）。Big-5只用於繁體中文，無法顯示日文或韓文。Unicode是國際化應用的最佳選擇。

多媒體數碼化與檔案格式

▼

圖像檔案格式

  • BMP：無壓縮點陣圖，檔案大
  • JPEG：有損壓縮，適合照片，不支援透明
  • PNG：無損壓縮，支援透明（alpha通道）
  • GIF：256色，支援動畫和1-bit透明

音頻檔案格式

  • WAV：無壓縮，高品質，檔案大
  • MP3：有損壓縮，檔案小，最常見

視頻檔案格式

  • AVI：容器格式，可含不同編碼
  • MPEG4 / MP4：高效壓縮，最常用

檔案大小計算公式

點陣圖大小 = 寬度(px) × 高度(px) × 色彩深度(bits) / 8 / 1024 / 1024 MB

音頻大小 = 採樣率(Hz) × 位元深度 × 聲道數 × 時間(秒) / 8 / 1024 / 1024 MB

電腦系統基礎 — 硬件、軟件與操作系統

CPU結構、記憶體系統、輸入輸出設備、系統軟件、操作模式

⚙️

CPU結構與功能

▼

CPU主要部件

  • 控制單元 (CU)：控制指令的提取、解碼和執行順序
  • 算術邏輯單元 (ALU)：執行算術和邏輯運算
  • 寄存器 (Register)：CPU內部高速暫存器
  • 快取記憶體 (Cache)：CPU與RAM之間的高速緩衝

提取-解碼-執行循環 (Fetch-Decode-Execute Cycle)

  1. 提取 (Fetch)：從記憶體提取下一條指令
  2. 解碼 (Decode)：CU解釋該指令
  3. 執行 (Execute)：ALU執行運算或傳輸數據
💾

記憶體系統

▼
類型易失性速度用途
RAM是（斷電丟失）極快 (ns)主記憶體，運行中的程式和數據
ROM否（斷電保留）快BIOS/UEFI韌體
Cache是最快CPU與RAM之間的緩衝
SSD否快 (0.1ms)輔助儲存
HDD否較慢 (5-10ms)輔助儲存（機械式）

⚠️ 常見錯誤：RAM是易失性的（volatile），斷電後數據會丟失。ROM是非易失性的（non-volatile），斷電後數據仍保留。

🖥️

操作系統 (OS) 功能

▼

操作系統是電腦最基礎的系統軟件，主要功能包括：

  • 記憶體管理：分配和釋放RAM給不同程式
  • 處理器管理：CPU排程（多任務處理）
  • 檔案管理：檔案和目錄的組織、存取控制
  • 輸入輸出管理：管理各種外圍設備
  • 提供用戶介面：GUI或CLI
  • 虛擬記憶體：使用硬碟空間模擬額外RAM
🔄

操作模式

▼
  • 批次處理 (Batch Processing)：收集一批工作後統一處理，不需用戶互動
  • 即時處理 (Real-time Processing)：立即處理輸入並產生輸出（如：航空訂票系統）
  • 並行處理 (Parallel Processing)：多個CPU同時處理不同部分
  • 分散式處理 (Distributed Processing)：多台電腦聯網共同處理
  • 虛擬化 (Virtualization)：在單一硬件上運行多個虛擬機器

互聯網及其應用 (一) — 網絡基礎與通訊協定

🌐

網絡類型：LAN vs WAN

▼
特性LAN (局域網)WAN (廣域網)
覆蓋範圍小（建築物/校園內）大（城市/國家/全球）
擁有權單一組織多個組織/服務供應商
速度快（通常100Mbps-10Gbps）較慢（取決於連接）
延遲低較高
例子學校電腦室網絡、家庭Wi-Fi互聯網、銀行ATM網絡
📡

網絡硬件

▼
  • 網卡 (NIC)：連接電腦到網絡
  • 交換機 (Switch)：在LAN內轉發數據幀（基於MAC地址）
  • 路由器 (Router)：在不同網絡之間轉發數據包（基於IP地址）
  • 數據機 (Modem)：調製和解調信號（模擬↔數碼轉換）
  • 無線接入點 (WAP)：提供Wi-Fi連接

💡 記憶：Switch用MAC（局域內），Router用IP（跨網絡）。

📨

TCP/IP 協議套件

▼

TCP (傳輸控制協議)

  • 確保數據可靠傳送
  • 數據分段和重組
  • 錯誤檢測和重傳
  • 流量控制

IP (互聯網協議)

  • 定址（IP地址）
  • 路由選擇（決定數據包的路徑）

IPv4：32位元，約43億個地址
IPv6：128位元，約3.4×10³⁸個地址

🔗

DNS、URL、HTTP/HTTPS

▼
  • DNS (域名系統)：將域名轉換為IP地址（如 www.google.com → 142.250.80.4）
  • URL (統一資源定位符)：標識網上資源的地址
  • HTTP：超文本傳輸協議，未加密
  • HTTPS：HTTP + SSL/TLS加密，提供機密性、完整性和身份驗證

URL結構：https://www.example.com:443/path/page.html?query=abc

雲端運算與物聯網

  • PaaS（平台即服務）：提供開發平台（如Google App Engine）

網絡安全威脅

▼
  • 病毒 (Virus)：自我複製，附著於檔案
  • 蠕蟲 (Worm)：自我複製，通過網絡傳播
  • 木馬 (Trojan)：偽裝成合法軟件的惡意程式
  • 勒索軟件 (Ransomware)：加密檔案，要求贖金
  • 間諜軟件 (Spyware)：秘密收集用戶資訊

  • 防火牆 (Firewall)：根據規則控制進出網絡流量
  • VPN：加密的虛擬專用網絡通道
  • WPA：Wi-Fi無線安全協議
  • 防毒軟件：檢測和移除惡意軟件
🔐

加密與認證

▼

對稱加密 vs 公開金鑰加密

  • 對稱加密：加密和解密使用同一金鑰，速度快（如AES）
  • 公開金鑰加密：一對金鑰——公鑰加密，私鑰解密（如RSA）

SSL/TLS與數碼證書

  • SSL/TLS使用公開金鑰加密進行初始密鑰交換
  • 然後使用對稱加密進行數據傳輸
  • 數碼證書由CA（證書頒發機構）簽發，驗證網站身份

計算思維與程式編寫 (一) — 基礎概念與編程實踐

🧠

計算思維的核心元素 (CT Elements)

▼

1. 抽象化 (Abstraction)

從複雜問題中提取關鍵特徵，忽略不相關細節，建立簡化模型。

💡 例子：設計地鐵路線圖時，只保留車站和連接線，忽略實際地理距離和彎曲——這就是抽象化。

2. 算法設計 (Algorithm Design)

設計一系列明確的、逐步的指令來解決問題。

算法的五個重要特徵：

  • 有限性 (Finiteness)：必須在有限步驟內結束
  • 明確性 (Definiteness)：每一步驟必須清楚無歧義
  • 輸入 (Input)：可有0個或多個輸入
  • 輸出 (Output)：至少產生一個輸出
  • 有效性 (Effectiveness)：每一步驟都是可行的基本操作

3. 自動化 (Automation)

將算法轉化為電腦能夠執行的程式碼，讓電腦自動完成任務。

⚠️ 考試重點：計算思維 ≠ 程式編寫。計算思維是解決問題的思維方式，程式編寫是實現工具。

偽代碼 (Pseudocode) 與三大控制結構

▼

偽代碼是用自然語言描述算法的方式，不依賴特定程式語言的語法。DSE考試中，偽代碼用於考核算法邏輯。

1. 順序結構 (Sequence)

輸入 x
輸入 y
sum = x + y
輸出 sum

2. 選擇結構 (Selection)

IF score >= 90 THEN
    輸出 "A"
ELSE IF score >= 80 THEN
    輸出 "B"
ELSE IF score >= 70 THEN
    輸出 "C"
ELSE
    輸出 "F"
ENDIF

巢狀選擇 (Nested IF)：

IF x > 0 THEN
    IF x > 10 THEN
        輸出 "Large positive"
    ELSE
        輸出 "Small positive"
    ENDIF
ELSE
    輸出 "Non-positive"
ENDIF

3. 迭代結構 (Iteration)

FOR 循環（已知次數）：

sum = 0
FOR i = 1 TO 10
    sum = sum + i
NEXT i
輸出 sum          // 結果 = 55 (1+2+...+10)

WHILE 循環（條件控制）：

count = 0
WHILE count < 5 DO
    輸出 count
    count = count + 1
ENDWHILE
// 輸出：0, 1, 2, 3, 4

REPEAT-UNTIL 循環（至少執行一次）：

REPEAT
    輸入 password
UNTIL password = "correct"
輸出 "Access granted"
⚠️ 考試陷阱：WHILE 先檢查條件再執行（可能0次），REPEAT-UNTIL 先執行再檢查（至少1次）。

程式實例：從偽代碼到 Python

▼

例子1：計算1到N的和

偽代碼：

輸入 N
sum = 0
FOR i = 1 TO N
    sum = sum + i
NEXT i
輸出 sum

Python：

N = int(input("輸入 N: "))
sum = 0
for i in range(1, N + 1):
    sum = sum + i
print(f"1到{N}的和為: {sum}")

例子2：找出最大值

偽代碼：

輸入 a, b, c
max = a
IF b > max THEN max = b ENDIF
IF c > max THEN max = c ENDIF
輸出 "最大值: ", max

Python：

a = int(input("輸入 a: "))
b = int(input("輸入 b: "))
c = int(input("輸入 c: "))
max_val = a
if b > max_val: max_val = b
if c > max_val: max_val = c
print(f"最大值: {max_val}")

例子3：判斷質數

n = int(input("輸入一個正整數: "))
is_prime = True
if n <= 1:
    is_prime = False
else:
    for i in range(2, int(n**0.5) + 1):
        if n % i == 0:
            is_prime = False
            break
if is_prime:
    print(f"{n} 是質數")
else:
    print(f"{n} 不是質數")
💡 考試重點：DSE 程式題目要求學生能閱讀偽代碼/流程圖推理結果，或將問題轉化為偽代碼。不需要寫出完整Python語法——偽代碼更注重邏輯而非語法精確性。

模組化程式設計 (Modular Programming)

▼

函數 (Function) vs 程序 (Procedure)

特性函數 (Function)程序 (Procedure)
返回值有（return value）無
調用方式result = sqrt(16)display_message()

Python函數實例

# 定義函數：計算矩形面積
def calculate_area(length, width):
    """返回矩形面積"""
    area = length * width
    return area

# 調用函數
room_area = calculate_area(5, 4)   # 結果 = 20
print(f"房間面積: {room_area} 平方米")

# 定義程序：顯示歡迎訊息
def show_welcome(name):
    """顯示歡迎訊息（不返回值）"""
    print(f"歡迎, {name}!")
    print("祝你有美好的一天！")

# 調用程序
show_welcome("Alice")

模組化的好處

  • 可重用性：同一函數可在多處調用，無需重複編寫
  • 可維護性：修改函數內部不影響調用它的程式
  • 可讀性：函數名稱描述其功能，程式邏輯清晰
  • 便於除錯：可單獨測試每個函數
  • 團隊協作：不同人負責不同模組
💡 DSE考試：參數傳遞（pass by value）——函數接收的是值的副本，修改參數不影響原變量。

陣列 (Array) 操作詳解

▼

一維陣列

# Python (list)
scores = [85, 92, 78, 90, 88]
print(scores[0])       # 輸出: 85 (第一個元素，索引=0)
print(scores[2])       # 輸出: 78 (第三個元素)
print(len(scores))     # 輸出: 5 (陣列長度)
print(scores[-1])      # 輸出: 88 (最後一個元素)

遍歷陣列

# 方法1：用索引
for i in range(len(scores)):
    print(f"學生{i+1}: {scores[i]}")

# 方法2：直接遍歷元素
total = 0
for score in scores:
    total = total + score
average = total / len(scores)
print(f"平均分: {average}")  # 輸出: 86.6

常用陣列操作（Python）

scores.append(95)           # 添加元素到末尾
scores.insert(2, 88)        # 在索引2處插入88
scores.pop()                 # 移除並返回最後一個元素
scores.remove(78)            # 移除第一個值為78的元素
scores.sort()                # 升序排序
scores.reverse()             # 反轉順序

二維陣列（矩陣）

# 3x3 矩陣
matrix = [
    [1, 2, 3],
    [4, 5, 6],
    [7, 8, 9]
]
print(matrix[0][1])    # 輸出: 2 (第0行第1列)
print(matrix[2][2])    # 輸出: 9 (第2行第2列)

# 遍歷二維陣列
for row in range(3):
    for col in range(3):
        print(matrix[row][col], end=" ")
    print()  # 換行

⚠️ 注意：Python 的 list 並非傳統意義上的固定長度陣列，但 DSE 考試中視為等價。偽代碼中的陣列索引也從 0 開始。

搜尋與排序算法 (Search & Sort)

▼

線性搜尋 (Linear Search)

逐一檢查每個元素，直到找到目標或遍歷完成。

arr = [5, 2, 8, 1, 9, 3]
target = 8
found = False
FOR i = 0 TO len(arr)-1
    IF arr[i] = target THEN
        輸出 "找到於位置 ", i
        found = True
    ENDIF
NEXT i
IF NOT found THEN 輸出 "未找到" ENDIF

時間複雜度：O(n) — 最壞情況需檢查所有元素

冒泡排序 (Bubble Sort)

重複比較相鄰元素，若順序不對則交換，每輪將最大元素「浮」到末尾。

arr = [5, 3, 8, 1, 2]
n = len(arr)
FOR i = 0 TO n-2                // 外循環：n-1 輪
    FOR j = 0 TO n-2-i           // 內循環：比較相鄰元素
        IF arr[j] > arr[j+1] THEN
            temp = arr[j]        // 交換
            arr[j] = arr[j+1]
            arr[j+1] = temp
        ENDIF
    NEXT j
NEXT i
// 結果：[1, 2, 3, 5, 8]
輪次(i)比較次數排序後狀態
04[3, 5, 1, 2, 8]
13[3, 1, 2, 5, 8]
22[1, 2, 3, 5, 8]
31[1, 2, 3, 5, 8]

時間複雜度：O(n²)

💡 記憶技巧：Bubble Sort 每輪結束時，最大的未排序元素會「冒泡」到正確位置——就像水中的氣泡上升一樣。

檔案處理 (File Handling)

▼

開啟與關閉檔案

' 偽代碼：
OPEN "data.txt" FOR INPUT AS #1     ' 開啟供讀取
OPEN "output.txt" FOR OUTPUT AS #2  ' 開啟供寫入
CLOSE #1                            ' 關閉檔案

Python 檔案操作

# 讀取整個檔案
with open("data.txt", "r", encoding="utf-8") as f:
    content = f.read()
    print(content)

# 逐行讀取
with open("data.txt", "r", encoding="utf-8") as f:
    for line in f:
        print(line.strip())  # strip() 移除換行符

# 寫入檔案
with open("output.txt", "w", encoding="utf-8") as f:
    f.write("第一行\\n")
    f.write("第二行\\n")

# 附加到檔案末尾
with open("log.txt", "a", encoding="utf-8") as f:
    f.write("新增的記錄\\n")

檔案模式

模式說明
"r"讀取（檔案必須存在）
"w"寫入（覆蓋原內容，不存在則建立）
"a"附加（在檔案末尾添加）

▼

三種錯誤類型

類型說明例子
語法錯誤 (Syntax Error)違反語言語法規則，程式無法執行prin("hello") # 拼寫錯誤

測試策略

  • 白盒測試 (White-box)：了解內部結構，測試所有執行路徑和邏輯分支
  • 黑盒測試 (Black-box)：不了解內部結構，僅根據規格測試輸入和輸出
  • 邊界值測試 (Boundary Value)：測試輸入範圍的邊界值（如 0, -1, 最大值+1）

測試值類別預期結果
0低於下限錯誤 / 拒絕
1下限接受
50正常範圍接受
100上限接受

除錯技巧

  • 逐步執行 (Step-through)：逐行執行程式，觀察變量變化
  • 加入偵錯輸出：在關鍵位置加入 print() 查看變量值
  • 追蹤表 (Trace Table)：手動追蹤每個變量在每步的值

私隱、數碼鴻溝與健康

▼

數碼鴻溝 (Digital Divide)

不同地區、社會經濟階層、年齡組別在ICT資源獲取和使用上的差距。體現在硬件獲取、互聯網接入、數碼技能等方面。

私隱與監控

  • 社交媒體數據收集
  • 網上行為追蹤（Cookies）
  • 政府監控和數據保留
  • 保障私隱方法：VPN、清除Cookies、強密碼、雙重驗證

健康影響

  • 重複性勞損 (RSI)
  • 眼睛疲勞和視力問題
  • 久坐導致的頸背疼痛
  • 預防：定時休息、人體工學設備、20-20-20法則
